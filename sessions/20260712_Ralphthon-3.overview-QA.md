리뷰 에이전트 논문 제출 가이드라인

본 문서는 2026년 7월 12일 진행된 해커톤 이벤트의 제출 가이드라인 Q&A 세션을 정리한 것이다. 이벤트는 ICML 주간 서울 행사(5회차)로, 테마는 **Auto Research**이며 AI 에이전트가 논문을 작성하고 리뷰하는 실험적 구조로 운영된다. Ralph Loop 진행 중에는 코딩 에이전트를 직접 조작할 수 없으며, 랩탑을 사용하려면 랍스터 코스튬을 착용해야 한다는 특별 규칙이 있다.

---

## 트랙 구성

이벤트는 두 개의 트랙으로 구성된다.

- **Track 1 (AI Scientist)**: AI 에이전트가 논문 형식의 연구 결과물을 생성하는 트랙.
- **Track 2 (Review Agent)**: Track 1의 논문들을 ICML 스타일로 리뷰하는 에이전트를 구축하는 트랙.

---

## 제출 규정

### PDF 논문 (Technical Report)

- 모든 참가자는 **ICML 포맷**의 Technical Report를 PDF로 작성하여 **16:30까지** Open Review 플랫폼에 제출해야 한다.
- **페이지 제한**: Main 텍스트 기준 **최대 4페이지** (하드 리밋).
  - 레퍼런스는 4페이지에서 **제외** (별도 허용).
  - Appendix는 **자유** (포함 여부 무관).
- 문서 구조(Structure)에 대한 별도 제한은 없으며 ICML 포맷 스타일을 따르면 된다.
- Track 2의 경우, Technical Report는 "리뷰 에이전트를 어떻게 만들었는지"를 설명하는 문서이다.

### 리뷰 제출 (Track 2)

- 16:30 제출 스냅샷 이후, 운영진이 **랜덤으로 논문 10편**을 배정한다.
- 배정된 10편 모두를 **30분 이내**에 리뷰 등록 완료해야 하며, 미완료 시 탈락 처리된다.
- 리뷰 양식은 **ICML 리뷰 양식**을 따르되, 점수 표기 위주로 작성하고 마지막에 Comment section을 추가한다.
- 리뷰는 PDF가 아닌, **리뷰 에이전트를 직접 실행**하여 나온 결과를 Open Review 플랫폼에 업로드하는 방식이다.

## 주요 Q&A 정리

| 질문 | 답변 |
|------|------|
| 레퍼런스도 4페이지에 포함되나? | 레퍼런스는 제외, Main 텍스트 4페이지 |
| Appendix도 포함되나? | Appendix는 자유, 제한 없음 |
| 리버설(Rebuttal)이 있나? | 이번 이벤트에는 리버설 없음 |
| 리버설 없으면 리뷰 작성 방식 변경? | 리버설 고려 불필요; 대신 논문 저자는 **셀프 리뷰** 필수 작성 |
| Track 2는 리뷰만 제출하면 되나? | Track 1·2 모두 Technical Report PDF 제출 + Open Review 플랫폼 사용 동일 |
| 리뷰를 PDF로 만들어야 하나? | 아님; 에이전트 실행 결과를 플랫폼에 직접 업로드 |
